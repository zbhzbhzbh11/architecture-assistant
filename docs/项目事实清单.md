## 一、项目概览

| 维度 | 事实 |
|------|------|
| 项目名称 | 架构风格智能助手 (Architecture Style Recommendation Assistant) |
| 项目类型 | 微服务架构推荐系统 |
| 微服务数量 | **7 个**（含 1 个 API 网关 + 5 个业务 Agent + 1 个知识库服务） |

| Python 版本 | 3.11 (Dockerfile 基准) |

## 二、服务清单（7 个微服务）

### 2.1 API Gateway（`services/api_gateway/`）
- **端口**: 8000
- **状态**: **已实现，完整**
- **核心端点**:
  - `POST /api/v1/recommend` — 主推荐编排（extract → match → evaluate → refactor）
  - `GET /health` — 健康检查
  - `GET /cache/stats` — 缓存统计
  - `POST /cache/clear` — 清空缓存
- **编排模式**: LangGraph StateGraph 优先；若 langgraph 未安装，fallback 到 httpx 顺序调用
- **工作流节点**: extract → match → evaluate → trace（4 节点顺序执行）
- **缓存**: 请求级缓存，支持 memory（默认）和 sqlite 两种后端，由 `CACHE_BACKEND` 环境变量切换
- **重构调用**: 在推荐完成后异步调用 refactoring-agent 获取重构建议，失败不阻塞主流程
- **关键文件**: `main.py` (226行), `langchain_workflow.py` (166行), `workflow_state.py` (36行)

### 2.2 Requirements Agent（`services/requirements_agent/`）
- **端口**: 8001
- **状态**: **已实现，完整**
- **核心端点**: `POST /extract`
- **实现方式**: 基于规则的中文关键词词典（10 个特征维度）提取架构特征
- **LLM 补充**: 当规则引擎命中维度 < 3 时，触发 LLM 语义补充（通过 OpenAI 兼容 API）
- **Few-shot**: 使用 `services/common/prompts/requirements_few_shot.py`（6 个示例），不可用时降级为零样本
- **否定过滤**: 检测"不需要"、"无高"等否定模式，避免误命中
- **关键文件**: `main.py` (272行)

### 2.3 Matching Agent（`services/matching_agent/`）
- **端口**: 8002
- **状态**: **已实现，完整**
- **核心端点**: `POST /match`
- **匹配引擎**:
  1. 规则引擎评分：特征标签基础分 + 学习权重加分 + 特定风格加成规则
  2. 图谱证据融合：从 knowledge-base `POST /graph/match` 获取证据，图谱加分上限为规则得分 50%
  3. 主流风格优先：Layered、Microservices、Event-Driven 始终出现在候选列表中
- **组合推荐**: 从 knowledge-base 获取组合定义，按"组件分数之和 + 特征覆盖加分 + 图谱互补加分 − 复杂度惩罚"排序
- **关键文件**: `main.py` (170行), `combo_matcher.py` (114行), `graph_matcher.py` (94行)

### 2.4 Evaluation Agent（`services/evaluation_agent/`）
- **端口**: 8003
- **状态**: **已实现，完整**
- **核心端点**: `POST /evaluate`
- **评估流程**:
  1. 按分数排序候选人
  2. 并行 LLM 调用：风格投票 + 自然语言摘要
  3. 混合推理：若 LLM 投票与规则选择不同，投票风格 +1 分
  4. 构建对比矩阵（含分数、优缺点、关键理由、拓扑图）
  5. 动态风险评估（Event-Driven/Microservices/Layered 有预定义条目，其余通用回退）
  6. 组合架构推荐写入 `recommended_combination`
  7. ADR 自动生成并写入 knowledge-base
- **Few-shot**: 使用 `services/common/prompts/evaluation_few_shot.py`（3 个示例）
- **Fallback**: LLM 不可用时使用 `_fallback_summary()` 模板化摘要
- **关键文件**: `main.py` (393行)

### 2.5 Knowledge Base（`services/knowledge_base/`）
- **端口**: 8004
- **状态**: **已实现，完整**
- **双后端架构**:
  - JSON 后端 (`json_repository.py`): 默认/fallback，无外部依赖
  - Neo4j 后端 (`graph_repository.py`): 可选，需 Neo4j 容器运行
  - 调度策略: `KNOWLEDGE_BACKEND` = `json` | `neo4j` | `auto`（优先 Neo4j，失败回退 JSON）
- **核心端点**:
  - `GET /styles` / `POST /styles` — 架构风格 CRUD
  - `GET /feedback` / `POST /feedback` / `GET /feedback/stats` — 反馈管理
  - `GET /feedback/weights` — 学习权重查询
  - `POST /graph/match` — 图匹配查询（Neo4j 不可用时返回 `available: false`）
  - `GET /graph/status` — 图数据库连接状态
  - `GET /combinations` — 架构组合查询
  - `POST /adr` / `GET /adr` / `GET /adr/{adr_id}` — ADR 持久化与检索
- **数据文件** (均在 `services/knowledge_base/data/`):
  - `architecture_styles.json` — 10 种架构风格
  - `architecture_combinations.json` — 5 种组合架构
  - `learned_weights.json` — 反馈驱动的学习权重
  - `feedback_log.json` — 反馈记录
  - `adr_records.json` — ADR 决策记录
- **关键文件**: `main.py` (228行), `graph_repository.py` (388行), `json_repository.py` (234行), `init/init_neo4j.py` (175行)

### 2.6 Refactoring Agent（`services/refactoring_agent/`）
- **端口**: 8005
- **状态**: **已实现，完整**
- **核心端点**: `POST /refactor`
- **实现方式**: 规则引擎主导（关键词检测 + 特征推断），LLM 可选润色
- **功能**:
  1. 坏味检测：5 种架构坏味（单体耦合、扩展瓶颈、发布缓慢、数据瓶颈、遗留系统）
  2. 模式选择：5 种重构模式（绞杀者模式、防腐层、模块化单体优先、CQRS 迁移、事件驱动迁移）
  3. 规则模板生成：迁移步骤 + 风险 + 缓解措施
  4. LLM 润色：可选，改进描述质量但保持结构不变
- **关键文件**: `main.py` (340行)

### 2.7 共享模块（`services/common/`）
- **状态**: **已实现，完整**
- **缓存模块** (`common/cache/`):
  - `simple_cache.py` — 内存缓存，TTL 支持，线程安全
  - `sqlite_cache.py` — SQLite 持久化缓存，接口兼容
  - `hash_utils.py` — 缓存键生成（SHA256），含知识库版本指纹
- **Few-shot 提示** (`common/prompts/`):
  - `requirements_few_shot.py` — 6 个示例，需求特征提取
  - `evaluation_few_shot.py` — 3 个示例，评估摘要生成

## 三、知识图谱 (Neo4j)

|------|------|

## 六、LLM 缓存模块

| 维度 | 事实 |
|------|------|
| 缓存粒度 | 请求级缓存（以 requirement + model + knowledge_version 为键） |
| 后端 | memory（默认，TTL 3600s）或 sqlite（持久化） |

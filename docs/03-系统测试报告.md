# 系统测试报告

## 1. 测试目的

验证"基于大模型的软件架构风格智能助手"系统是否满足需求规格说明书中定义的功能需求、AI 系统特有需求和非功能需求，确保系统具备以下能力：

1. 从自然语言需求中正确提取 10 维非功能特征。
2. 基于知识库和规则引擎推荐至少 3 种候选架构风格。
3. 生成包含推荐理由、对比矩阵、风险建议的最终报告。
4. 在 LLM 不可用时自动降级，保持核心推荐链路可用。
5. Web 前端正常展示推荐结果、对比矩阵和拓扑图。

---

## 2. 测试范围

| 测试项 | 说明 |
|--------|------|
| 需求解析 | 关键词匹配命中率、否定语义过滤、LLM 语义补全 |
| 架构推荐 | Top 3 完整性、主流架构覆盖率、评分规则正确性 |
| 决策支持 | 最终推荐产出、对比矩阵完整性、决策可解释性、风险与建议 |
| 知识库 | 架构风格数量（≥10）、字段完整性、扩展接口 |
| LLM 集成 | LLM 投票、LLM 摘要、LLM 降级 |
| Web 前端 | 结果展示、矩阵表格、Mermaid 拓扑图渲染 |
| 异常处理 | 网关 502、前端错误恢复 |
| Agent 协作 | Pipeline 链路的端到端通过率 |

## 4. 测试方法

### 4.1 测试层次

```
┌─────────────────────────────────┐
│        冒烟测试 (Smoke Test)      │  快速验证全链路连通性
│        run_smoke.py              │  20 条用例串行执行
├─────────────────────────────────┤
│        回归测试 (Regression)       │  结构完整性 + 指标统计
│        run_regression.py         │  输出 JSON + Markdown 报告
├─────────────────────────────────┤
│        单元测试 (Unit Test)        │  函数级逻辑验证
│        pytest tests/unit/ -v      │  23 条用例, 4 个模块
└─────────────────────────────────┘
```

### 4.2 回归测试指标定义

| 指标 | 计算方法 | 目标 |
|------|---------|------|
| Top3 完整率 | candidates 数组长度 ≥ 3 的用例占比 | 100% |
| 主流架构覆盖率 | candidates 中包含 Layered/Microservices/Event-Driven 至少 1 种的用例占比 | 100% |
| 最终推荐产出率 | recommended_style 不为空的用例占比 | 100% |
| 决策可解释率 | decision_basis.rule_engine 存在且 llm_summary 字段存在的用例占比 | 100% |
| 对比矩阵产出率 | comparison_matrix 行数 ≥ 3 的用例占比 | 100% |
| 总通过率 | 以上 5 项全部满足的用例占比 | ≥ 95% |

### 4.3 单元测试覆盖模块

| 测试文件 | 用例数 | 覆盖模块 |

## 17. 缺陷与改进建议

### 17.1 已知局限（非缺陷，为设计边界）

| 编号 | 局限 | 影响 | 建议 |
|------|------|------|------|
| L-01 | 知识库为 JSON 文件存储 | 并发写入安全、大规模查询性能受限 | 后续升级为 Neo4j 或 PostgreSQL |
| L-02 | 规则权重硬编码 | 无法动态调整评分策略 | 抽取为配置文件或环境变量 |
| L-03 | 仅 3 种核心风格有针对性风险建议 | 其余 7 种使用通用模板 | 补充完整的 STYLE_RISK_MAP |
| L-04 | 案例学习为数据收集阶段 | 无自动权重学习闭环 | 实现基于反馈数据的周期性权重更新 |
| L-05 | 网关串行调用 Agent | 时延累加 | 将独立步骤并行化，或引入缓存 |
| L-06 | 无请求级缓存 | 相同需求重复计算 | 在网关层增加 dict 缓存 |

### 17.2 改进建议（按优先级排列）

**第一优先（建议近期完成）：**
1. 为全部 10 种架构风格补充针对性的风险与缓解建议。
2. 将评分权重 (tag 2 分 / extra 1 分 / llm_vote 1 分) 抽取为可配置参数。

**第二优先（后续迭代）：**
3. 基于 feedback_log.json 的统计数据，实现轻量级权重调整机制。
4. 在网关层增加基于需求文本 hash 的请求级缓存。
5. 对 evaluation-agent 中 LLM 投票和摘要的并行执行效果进行量化验证。

**第三优先（远期优化）：**
6. 升级知识库为 Neo4j 图数据库，支持风格间关系推理。
7. 增加 ADR（架构决策记录）自动生成功能。
8. 引入消息队列（如 Redis）实现 Agent 间异步通信。

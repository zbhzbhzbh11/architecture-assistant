---

## 1. 测试目的

验证"架构风格智能助手"系统是否满足需求规格说明书（01-需求规格说明书.md）中定义的 14 项功能需求（FR）、9 项 AI 系统特有需求（AIR）和 8 类非功能需求（NFR）。核心验证目标：

1. 从自然语言需求中正确提取 10 维非功能特征（含否定语义过滤、LLM 语义补全）。
2. 基于规则引擎 + 知识图谱 + LLM 三层驱动，推荐至少 3 种候选架构风格。
3. 生成包含决策依据、对比矩阵、风险建议、ADR 记录的完整推荐报告。
4. 支持架构组合推荐（5 种预定义组合）和架构重构建议（5 坏味 + 5 模式）。
5. LLM / Neo4j / LangGraph 不可用时自动降级，核心推荐链路不中断。
6. Web 前端 11 个展示区块正常渲染。

### 6.2 跳过原因说明

3 条跳过的测试均属于 `test_knowledge.py::TestGraphRepository` 类：

| 跳过用例 | 跳过原因 | 标记 |
|---------|---------|------|
| `test_graph_status_neo4j` | 需要 Neo4j 运行环境 | `@pytest.mark.integration` |
| `test_get_styles_from_neo4j` | 需要 Neo4j 运行环境 | `@pytest.mark.integration` |
| `test_graph_status_has_counts` | 需要 Neo4j 运行环境 | `@pytest.mark.integration` |

跳过机制：`pytest` 通过 `conftest.py` 注册了 `integration` 标记。运行 `pytest tests/unit/ -v` 时未指定 `-m integration`，这三条 Neo4j 集成测试自动跳过。启动 Neo4j 后运行 `pytest tests/unit/ -v -m integration` 即可执行。

## 21. 性能与响应时间

| 含 LLM 模式 | 6-11s | 含 DeepSeek API 网络延迟 |
| 单 Agent 调用超时 | 15s/20s/25s | extract=15s, vote=20s, summary=25s |
| 网关总超时 | 30s | httpx.AsyncClient timeout |

**关键设计**:
- LLM 投票和摘要通过 `asyncio.gather` 并行执行，减少串行等待
- 请求级缓存避免相同需求的重复 LLM 调用
- 纯规则模式不受外部 LLM API 延迟影响

## 这个项目是什么

> **用 14 节课 + 108 道面试题 + 8 张架构图，把一个软件体系结构课程大作业，包装成能在保研/求职面试中自信讲述的完整项目。**

[Architecture Assistant](https://github.com/zbhzbhzbh11/architecture-assistant) 是一个 LLM + 知识图谱双驱动的架构风格推荐系统。用户输入自然语言软件需求，系统自动完成特征提取、风格匹配和评估决策，输出包含推荐理由、对比矩阵和风险分析的完整报告。

本学习指南是 Architecture Assistant 的**系统化学习教程 + 面试备战手册**，专为以下人群设计：

- **课程答辩者** —— 14 节课带你从项目背景讲到测试验收，答辩 25 分钟全程有稿
- **保研/求职面试者** —— 108 道面试题 + STAR 面试稿 + 4 个版本简历 Bullet
- **想深入理解 Compound AI 的开发者** —— 每节课都有源码级对照和架构图

### 本指南的特色

| 特色 | 说明 |
|------|------|
| **108 道面试题** | 覆盖项目介绍、架构设计、Agent/LLM、Neo4j、规则引擎、测试工程、简历追问 7 大方向 |

## 技术亮点表格

| 亮点 | 说明 | 证据文件 |
|------|------|---------|
| **Compound AI System** | 规则引擎主导 + Neo4j 图谱推理 + LLM 语义增强，三层协作 | graph_matcher.py + evaluation-agent |
| **全链路 LLM 降级** | 8 个 LLM 调用点各配备独立 fallback，LLM 全关也能正常运行 | 各 agent main.py 的 try/except |
| **LangGraph 双引擎** | StateGraph 优先，ImportError/运行时异常自动回退手动编排 | langchain_workflow.py |
| **Neo4j + JSON 双后端** | 6 节点 + 6 关系图模型，三模式（json/neo4j/auto）透明切换 | graph_repository.py + json_repository.py |

| **规则引擎 7 条规则** | 标签 +2/规则 +1/学习 +1，每分可追溯 | matching_agent score_style() |
| **LLM 双后端缓存** | 内存/SQLite，SHA256 键 + knowledge_version 自动失效 | simple_cache.py + sqlite_cache.py |
| **ADR 决策溯源** | 每次推荐自动生成 ADR-YYYYMMDD-NNN，双存储 | json_repository.py add_adr() |
| **架构组合推荐** | 5 种模式，4 因素评分公式，复杂度惩罚 | combo_matcher.py |
| **架构重构建议** | 5 种坏味 + 5 种模式，LLM 润色可选 | refactoring-agent |
| **四层测试体系** | 76 单元 + 20 回归 × 5 维 + 冒烟 + 43 验收 | tests/ + scripts/ |

## 项目可讲亮点

### 答辩/面试中最有说服力的 5 个点

1. **"规则引擎主导，LLM 辅助"** —— 不是调 API 的聊天机器人。每分可追溯，LLM 投票只加 1 分。
2. **"LLM 不可用时系统照常运行"** —— 8 个降级点，LLM 全关后 20 条回归全部通过。
3. **"6 个微服务 + 双引擎编排"** —— 系统自身架构就是课程内容的实践。LangGraph 和手动编排功能等价。
4. **"Neo4j 可用时增强推理，不可用时 JSON fallback"** —— 务实的外部依赖管理。
5. **"43 项验收 100% 通过 + 四层测试"** —— 不只是"能跑"，而是"能证明跑对了"。

## 当前不足与后续演进

> 诚实标注，面试被问到时可以大方承认并说出改进方向。

| 不足 | 改进方向 |
|------|---------|
| 知识库仅 10 种风格，手工维护 | 知识库管理界面 + 自动化知识获取 |
| 特征提取依赖关键词词典 | 探索轻量级分类模型或 Embedding 相似度匹配 |
| 学习权重冷启动（仅 3 条反馈） | 降低初始阈值或引入合成反馈 |
| 重构检测基于关键词而非代码分析 | 引入 AST 分析或架构度量指标 |
| 无 CI/CD、无压力测试、无 API 认证 | GitHub Actions + Locust + API Key |
| LangGraph 仅使用线性 DAG | 探索条件路由和并行节点 |

详见 [项目复盘报告 §8-§9](项目复盘报告.md#8-当前不足)。

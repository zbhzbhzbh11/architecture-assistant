## 8. 修复建议

### 8.1 环境相关

| 问题 | 建议 | 优先级 |
|---|---|---|
| Docker 不可用 | 安装 Docker Desktop 或使用 docker-compose 替代方案（如 podman）。`docker-compose.yml` 已验证配置正确 | P1 |
| pip 代理拦截 | 配置 pip 代理：`pip config set global.proxy http://proxy:port` 或使用 `--proxy` 参数；或在能出网的环境中运行 | P1 |
| httpx 走代理 | **永久修复：** 在 `api-gateway/app/main.py` 和 `evaluation-agent/app/main.py` 的 `httpx.AsyncClient` 中增加 `trust_env=False`，避免受系统代理干扰 | P0 |

### 8.2 代码相关

| 问题 | 建议 | 文件 | 优先级 |
|---|---|---|---|
| .env 未自动加载 | 在 `evaluation-agent/app/main.py` 顶部增加 `from dotenv import load_dotenv; load_dotenv()` 或使用 `python-dotenv` 包 | `services/evaluation_agent/app/main.py` | P1 |
| LLM 调用串行 | `llm_vote_style()` 和 `llm_summary()` 可并行调用（两者无依赖关系），使用 `asyncio.gather()` 可减少 ~40% 时延 | `services/evaluation_agent/app/main.py:112-127` | P2 |
| 无请求缓存 | 相同需求重复调用浪费 LLM token。增加简单 dict 缓存（key=requirement hash） | `services/api_gateway/app/main.py` | P2 |

### 8.3 启动脚本化建议

将手动启动命令整合为 `start.sh`：

```bash
#!/bin/bash
export NO_PROXY="localhost,127.0.0.1"
export no_proxy="localhost,127.0.0.1"
export REQUIREMENTS_AGENT_URL="http://localhost:8001"
export MATCHING_AGENT_URL="http://localhost:8002"
export EVALUATION_AGENT_URL="http://localhost:8003"

# Load .env if exists
if [ -f ".env" ]; then
  export $(grep -v '^#' .env | xargs)
fi

echo "Starting services..."

python -m uvicorn services.knowledge_base.app.main:app --host 0.0.0.0 --port 8004 &
python -m uvicorn services.requirements_agent.app.main:app --host 0.0.0.0 --port 8001 &
python -m uvicorn services.matching_agent.app.main:app --host 0.0.0.0 --port 8002 &
python -m uvicorn services.evaluation_agent.app.main:app --host 0.0.0.0 --port 8003 &
sleep 4
python -m uvicorn services.api_gateway.app.main:app --host 0.0.0.0 --port 8000 &
sleep 2

echo "All services started."
echo "  Frontend: http://localhost:3000"
echo "  API Docs: http://localhost:8000/docs"
```

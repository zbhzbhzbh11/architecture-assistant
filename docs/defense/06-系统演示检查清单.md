## ⚠ Windows 用户注意

本文档中的命令已适配 Windows (cmd/PowerShell) 和 Unix (bash/zsh) 双平台。

**关键差异速查**:

| 场景 | Windows (cmd) | Unix (bash) |
|------|--------------|-------------|
| JSON 引号 | `-d "{\"key\":\"val\"}"` (双引号+转义) | `-d '{"key":"val"}'` (单引号) |
| 续行符 | `^` | `\` |
| 丢弃 stderr | `2>NUL` | `2>/dev/null` |
| 端口检查 | `netstat -ano \| findstr <port>` | `netstat -ano \| grep <port>` |
| 过滤文本 | `findstr <word>` | `grep <word>` |

本文档命令已统一写成**单行 + 双引号 JSON**，在 Windows cmd 和 Unix bash 下均可直接粘贴运行。

## 十二、失败应急方案

### 12.1 Docker 完全不可用

**症状**: Docker Desktop 无法启动或 `docker compose up` 失败。

**应急步骤**:
1. 准备好在 IDE 中展示的关键代码文件和架构图
2. 打开 `docs/02-架构设计文档.md` 中的 C4 图（Mermaid 源码 + 说明）
3. 展示 `scripts/check_assignment.py` 的执行结果截图
4. 展示单元测试和回归测试的执行结果截图
5. 口头说明系统设计，不依赖实时演示

**已准备材料**:
- [ ] `docs/02-架构设计文档.md` 已在 IDE 中打开
- [ ] `docs/defense/05-C4模型与UML图.md` 已在 IDE 中打开
- [ ] 关键代码文件（6 个 Agent main.py）已打开

### 12.2 服务部分启动失败

**症状**: 某个容器反复 restarting 或 exited。

```bash
# 快速诊断
docker compose ps
docker compose logs <failed-service> --tail 20
```

| 失败服务 | 影响 | 应急 |
|---------|------|------|
| frontend | 无法浏览器演示 | 用 curl + Swagger 展示 API |
| api-gateway | 完全无法演示 | 重启该服务：`docker compose restart api-gateway` |
| requirements-agent | 网关返回 502 | 重启该服务 |
| matching-agent | 网关返回 502 | 重启该服务 |
| evaluation-agent | 网关返回 502 | 重启该服务 |
| knowledge-base | 网关返回 502 | 重启该服务 + Neo4j |
| refactoring-agent | 重构建议为空 | 主推荐正常，口头说明 |
| neo4j | 图谱证据为空 | JSON fallback 生效，口头说明 |

### 12.3 LLM 调用全部超时

**症状**: 响应等待 > 25 秒或返回 HTTP 错误。

**应急**: 临时清空 `.env` 中的 LLM 配置，重启服务，走纯规则模式。

```bash
# 备份 .env
cp .env .env.bak
# 临时注释 LLM 配置
# 或在 docker-compose 中覆盖为空
LLM_API_KEY="" docker compose up -d
```

**话术**: "LLM 服务当前网络延迟较高，我切换到纯规则模式展示。系统设计了三级降级——纯规则模式下推荐结论、对比矩阵、风险建议全部完整。"

### 12.4 前端渲染异常

**症状**: Mermaid 图不显示，或 marked.js 渲染失败。

**应急**: 
1. F12 打开控制台查看 CDN 加载失败的具体 URL
2. 如果 CDN 不可达（网络问题），刷新页面重试
3. 如果 CDN 确实不通，直接用 curl 展示 JSON 响应，口头解释各字段

### 12.5 推荐结果与预期不符

**症状**: 首选推荐不是 Event-Driven，或得分与预期有差异。

## 附录: 一键检查脚本

```python
#!/usr/bin/env python3
"""系统演示环境一键检查 — Windows / Unix 通用"""
import subprocess, json, sys, os

def run(cmd, shell=True):
    try:
        r = subprocess.run(cmd, shell=shell, capture_output=True, text=True, timeout=30)
        return r.stdout.strip() or r.stderr.strip(), r.returncode
    except Exception as e:
        return str(e), 1

def check(label, cmd, pipe_cmd=None):
    print(f"{label}:")
    out, code = run(cmd)
    if code != 0:
        print(f"  FAIL ({out})")
        return
    if pipe_cmd:
        out, code = run(pipe_cmd, shell=True)
    print(f"  {out[:200]}")
    print()

print("=== 系统演示环境检查 ===\n")

check("1. Docker 容器状态", "docker compose ps")
check("2. API Gateway 健康检查", "curl -s http://localhost:8000/health")
check("3. 前端可访问性", 'curl -s -o NUL -w "%{http_code}" http://localhost:3000')
```
